CSS隐藏元素的方法

发表于 2021-06-15 更新于 2021-09-19 分类于 前端 阅读次数：

面试的时候会被问到 CSS 隐藏元素的方式以及每种方式的点击事件。在这里总结一下相关的信息。

对于隐藏元素我能想到的有以下几种方案，如下

  • display: none
  • visibility: hidden
  • opcity: 0
  • overflow: hidden
  • position 定位
  • z-index 层级设置

display 方案

可以设置 display:none，彻底的隐藏元素，从而不占据空间大小，也无法响应事件。

visibility 方案

对于 visibility 来说有 visible，hidden 等取值，显示或隐藏元素而不更改文档的布局。
对于 hidden 来说隐藏元素，但是其他元素的布局不改变，相当于此元素变成透明，占据空间大小且无法触发响应事件

opcity 方案

opacity 属性指定了一个元素的不透明度。opacity 属性指定了一个元素后面的背景的被覆盖程度。一个元素和它包含的子元素都会具有和元素背景相同的透明度，除非它的子元素有小于这个元素的 opacity 属性值。
对于 opcity 设置为 0 来说，能够实现隐藏元素的效果，占据空间大小且能够触发响应事件

overflow 方案

CSS 属性 overflow 定义当一个元素的内容太大而无法适应块级格式化上下文的表现。
为使 overflow 有效果，块级容器必须有一个指定的高度（height 或者 max-height）或者将 white-space 设置为 nowrap。
对于 overflow: hidden 来说隐藏元素溢出部分，占据空间且无法响应点击事件。

position 方案

对于 position 来说 fixed/absolute/relative 都能够实现隐藏元素的效果。思路都是解决将元素移出可视区域。
对于 position: fixed/absolute 来说，不占据空间大小且无法点击
对于 position: relative 来说，占据空间大小且无法点击

z-index 方案

z-index 属性设定了一个定位元素及其后代元素或 flex 项目的 z-order。当元素之间重叠的时候， z-index 较大的元素会覆盖较小的元素在上层进行显示。
z-index 值使其它元素遮盖该元素也算是一种隐藏。不占据空间大小且无法点击

❗ 思考：在此环境下，如何能够出发 z-index 被遮盖的元素? 在遮盖层设置 pointer-events:none,元素永远不会成为鼠标事件的 target
